Rotas (endpoints)

Caminhos web para obtenção e persistência dos dados.

Primeiro API consulta o banco full data raw, caso não encontrado consulta a API Agrotools tanto para analysis quanto para list territories.

Após obtenção dos dados é feita a junção das partes, persistindo no banco (caso não tenha erro) e retornado ao usuário.

As mensagens de erros obtidos na Agrotools são enviados nos campos error_territories e error_analysis do JSON de resposta, podendo aparecer em um ou ambos. Nesse caso o dado é retornado ao usuário mas não persistido no banco.

A API não distingue erros via Agrotools, retornando 200 nesses casos. Retornará um código diferente de 200 caso haja algum erro interno (500) ou de autenticação (403).

Os itens enviados (body) durante a requisição correspondem aos códigos dos territórios que se deseja consultar.

Abaixo segue um exemplo para uma das rotas via linguagem python:

from requests import get

url_base = "http://DOMINIO/api/v1/full_data_stage"

header = {"Content-Type":  "application/json",
          "authorization": "Bearer TOKEN"}

data = {"cdTarget": [8434559999985070, 8434551183268285070]}

response = get(url_base, json=data, headers=header)


Cd Territory

Requisição para obter os códigos dos territórios.


GET

Requisição

Method URL
GET /api/v1/cd_territory
Header Contents
Content-Type application/json
authorization Bearer TOKEN


Resposta

Status Response Body
200 {"cd_territory" : [ {...}, ..., {...} ]} 
{
    "cd_territory": [
      {'cd_territory': 8434551183268285070},
      {'cd_territory': 8434551183268285070},
      {'cd_territory': 596833845656168742}
    ]
}


Detalhes

Retorna todos os códigos presentes no banco de dados da aplicação.


POST

Requisição

Method URL
POST /api/v1/cd_territory
Header Contents
Content-Type application/json
authorization Bearer TOKEN
Request body Contents
cdTarget [ 8434551183268285070 ]


Resposta

Status Response Body
200 {"cd_territory" : [ {...}, ..., {...} ]} 
{
    "cd_territory": [
      {'cd_territory': 8434551183268285070}
    ]
}


Status Response Body
409 {"message" : 'Um ou mais códigos já existem.'}


Detalhes

Essa rota adiciona o código enviado ao banco de dados. Não permite códigos duplicados.

O erro 409 indica que alguém está tentando inserir um ou mais códigos já existentes no banco de dados.

Essa rota consulta o banco de dados da aplicação retornando últimos registros sem forçar atualização.

Detalhes

Os campos entre cd_territory e area_fazenda_calculada são referentes aos dados list territories. Já dataConsulta a soma_area_prodes_ha e houve_mudanca vêm de analysis.

Os campos "error_territories" e "error_analysis" são vinculados a cada parte dos dados.

Existem alguns serviços de processamento de dados (stage) atuando nessa rota quando o código não existe no banco:

  1. organização dos dados brutos de list territories

  2. organização dos dados brutos de analysis

  3. merge de territories e analysis

A rota sempre retorna o território com o campo dataConsulta mais recente.

A geração do campo houve_mudanca é a comparação dos critérios de legalidade (nmCriteria). Caso um critério não seja igual ao do último registro do mês anterior, então houve_mudanca é True.
